a) La commande "git fetch" récupère les dernières modifications du dépôt distant vers votre dépôt local sans les fusionner avec votre code actuel, tandis que la commande "git pull" effectue un "git fetch" suivi d'un "git merge" pour fusionner les modifications téléchargées avec votre code actuel.
b) Pour annuler les modifications locales non validées dans Git, on peututiliser la commande "git reset". La commande "git reset --hard" annulera toutes les modifications locales non validées et réinitialisera le code à l'état actuel du dernier commit.
c) Pour supprimer une branche distante dans Git, on peut utiliser la commande "git push [nomdurepote] --delete [nomdelabranche]". Cette commande supprime la branche du dépôt distant. on peut également utiliser la commande "git branch -d [nomdelabranche]" pour supprimer une branche locale.
) Pour voir l'historique des modifications pour un fichier spécifique dans Git, on peut utiliser la commande "git log [nomdufichier]". Cette commande affiche toutes les modifications apportées au fichier, avec les messages de commit correspondants. on peut également utiliser la commande "git blame [nomdufichier]" pour afficher chaque ligne du fichier avec le nom et la date du dernier commit qui a modifié cette ligne.
